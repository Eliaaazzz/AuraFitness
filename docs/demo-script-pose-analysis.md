# 🎬 AI姿势分析功能演示脚本

## 演示场景: 深蹲动作分析

### 准备工作
1. 确保后端服务运行在 `http://localhost:8080`
2. 准备一张深蹲图片 (可以是手机自拍)
3. 获取OpenAI API Key并配置到 `.env`

---

## 演示流程 (5分钟)

### 场景设定
```
用户小王是健身新手，今天在家练深蹲。
他不确定自己的姿势是否正确，担心受伤。
于是用手机拍了一张照片，上传到我们的App。
```

### Step 1: 用户上传图片 (30秒)

**操作**: 打开移动App，进入"姿势分析"页面

```
1. 点击"拍照"按钮
2. 对准镜子，做深蹲姿势
3. 拍照
4. 选择训练类型: "深蹲"
5. 点击"开始AI分析"
```

**后台发生的事情**:
```javascript
// 前端代码 (PoseAnalysisScreen.tsx)
```

### Step 2: AI分析过程 (10-15秒)

**屏幕显示**: 
```

正在检测姿势问题
```

**后端处理流程**:
```java
// PoseAnalysisService.java
1. 接收图片 → base64编码
2. 构建提示词:
   "请分析这个深蹲动作的姿势，重点评估:
    1. 膝盖是否内扣或外翻
    2. 背部是否挺直
    3. 深蹲深度是否达标
    4. 重心是否稳定
    5. 脚掌位置是否正确"
    
3. 调用 GPT-4 Vision API
4. 解析返回结果
5. 保存到数据库 (workout_session + pose_analysis_result)
```

## 观众可能的疑问 (Q&A准备)

### Q1: "AI准确吗？会不会误判？"

**回答**:
```
好问题！我们做了两层保障:

1. GPT-4V本身准确率很高 (OpenAI官方数据>90%)
2. 我们会加入MediaPipe骨骼检测，提供数值化数据
   (比如"膝盖内扣15度"而不是模糊的"有点内扣")
3. 用户可以标记"分析不准确"，我们持续优化提示词

目前测试版准确率约85%，
正式上线前会达到90%+。
```

### Q2: "成本怎么控制？GPT-4V很贵吧？"

**回答**:
```
是的，GPT-4V大约 $0.01/图片 (1分钱一次分析)。

但我们的定价是 $19/月，
即使用户每天分析一次，也有 (19 - 30*0.01) = $18.7 利润。

而且我们会:
1. 用MediaPipe先做初筛 (免费)
2. 缓存相似姿势的分析结果
3. 只把需要深度分析的传给GPT-4V

预计实际成本能降到 $0.005/次。
```

### Q3: "为什么用户要付费？免费App那么多"

**回答**:
```
关键差异: 我们提供的是"专业价值"，不是"内容搬运"。

免费App只是YouTube视频合集，
用户看完还是不知道自己做得对不对。

我们提供的是:
- 个性化分析 (针对你的姿势)
- 量化评分 (清晰的进步指标)
- 具体建议 (怎么改，练什么肌肉)

这相当于线上私教，
真人私教 $50-100/小时，
我们 $19/月无限次，
价值感很明显。

而且我们看到:
- 健身新手愿意付费学正确姿势 (怕受伤)
- 企业愿意付费给员工福利 (提升留存率)
```
